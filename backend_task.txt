Завдання для створення бекенду для BunkerMuz Gallery

Фронтенд проекту створений як React додаток, він не вимагає серверного рендерингу та роботи із шаблонами.
Натомість потрібно створити моделі даних і методи доступу до них

1.  Створити проект на Django
2.  Створити аплікацію "api", яка буде обслуговуватись по адресі "api/".
3.  Створити і налаштувати папку для завантажених картинок "media"
4.  Створити і налаштувати папку для статичних файлів. 
    Вона має обслуговуватись по адресі ("/static/").
    В папку статики потрібно скопіювати вміст папки "build", це і є фронтент додаток
5.  Створити моделі даних з наступними полями:

    Для зберігання авторів "Author":
        "image" - ImageField
        "name" - CharField
        "title_text" - TextField
        "history" - TextField

    Для групованого виводу авторів "AuthorsGroup":
        "type" - !! поле вибору типу групи:
            1 - група із 3-х квадратних фотографій  ( ооо )
            2 - група із 3-х прямокутних фотографій ( ||| )
            3 - група із 2-х прямокутних фотографій ( - - )
        "author1" - посилання на автора
        "author2" - посилання на автора
        "author3" - посилання на автора

    Для картини "Paint":
        "image" - ImageField
        "author" - посилання на автора
        "title" - CharField
        "material" - CharField
        "size" - CharField
        "price" - CharField
        "sold" - BoolField

    Для даних контактів "CustomData":
        "title" = CharField
        "custom_key" = CharField
        "custom_value" = CharField

    5.1 Заповнити "CustomData" такими даними в "custom_key" і "custom_value":
        "address", "м. Тернопіль бульвар Шевченка, 1"
        "phone", "+38 098 295 1466"
        "email", "bunker@gmail.com"
        "facebook_page", "https://facebook.com"
        "instagram_page", "https://instagram.com"

6.  Створити API для видачі даних для фронтенду. View має повертати дані у форматі JSON.
    Зверніть увагу, тут наведеній повний шлях до методів, а ваша аплікація вже працює по адресі "api/"
    Потрібно наступні методи:

        /api/paints/getstart
        /api/paints/get/<from_number>/<count>
        /api/paints/detail/<paint_id>
        /api/paints/byauthor/<author_id>
        /api/authors/getstart
        /api/authors/get5
        /api/authors/get/<from_number>/<count>
        /api/authors/detail/<author_id>
        /api/custom/getdata
